Repeatable Project Steps (Recommended 1000 hours)

  1. Get a simple idea from some place like kaggle or make it up on your own.
  2. Choose a datastore.
  3. Model the data with a diagram
  4. Build the model in your program (Remove redundancy, PK, FK, indexes, proper data types)
  5. Find how to create test data for unit tests later
  6. How to load data into your model
  7. Automate producers for the models
  8. Build consumers for the models (Including aggregation functions) (Optional UI)
  9. Automate unit tests using the test data
  10. Data Integrity Scripts and alerts
  11. Performance tuning
  12. Monitor Scheduled ETL Jobs
  13. Documentation in README (Title, executive summary, example, features, installation, support, FAQ)

Nice to Know about Data Store

  1. Backup data and test restoring data
  2. How to alter/delete tables and columns in a datastore
  3. How to do advanced level CRUD for this datastore.
  4. For loops, procedures, user defined functions, and triggers in the datastore.
  5. Queries across two databases in the datastore
  6. Programmatically view the Information_Schema
  7. Creating and authorizing Database Users
