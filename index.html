Repeatable Project Steps (Recommended 1000 hours)

  1. Get a simple idea from some place like kaggle or make it up on your own.
  2. Choose a datastore.
  3. Backup data and test restoring data
  4. How to alter/delete tables and columns in a datastore
  5. How to do advanced level CRUD for this datastore.
  6. For loops, procedures, user defined functions, and triggers in the datastore.
  7. Queries across two databases in the datastore
  8. Programmatically view the Information_Schema
  9. Creating and authorizing Database Users
  10. Model the data with a diagram
  11. Build the model in your program (Remove redundancy, PK, FK, indexes, proper data types)
  12. Find how to create test data for unit tests later
  13. How to load data into your model
  14. Automate producers for the models
  15. Build consumers for the models (Including aggregation functions) (Optional UI)
  16. Automate unit tests using the test data
  17. Data Integrity Scripts and alerts
  18. Performance tuning
  19. Monitor Scheduled ETL Jobs
  20. Documentation in README (Title, executive summary, example, features, installation, support, FAQ)
